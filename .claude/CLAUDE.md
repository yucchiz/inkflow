# .claude/ 設定ガイド

Claude Code の動作設定を管理するディレクトリ。

## 構成

| ディレクトリ/ファイル | 役割                                     |
| --------------------- | ---------------------------------------- |
| `settings.json`       | 権限設定・フック                         |
| `rules/`              | glob パターンで自動適用される実装規約    |
| `agents/`             | 専門タスクを自律実行するエージェント定義 |
| `skills/`             | `/` コマンドで呼び出す定型タスク         |

## ルート CLAUDE.md との分担

- `/CLAUDE.md`: プロジェクト全体の方針（命名規則、禁止事項、リポジトリマップ）
- `/docs/architecture.md`: 技術詳細（React 19・TypeScript・Vite・IndexedDB・PWA 構成）
- `rules/`: 特定ファイル種別向けの詳細規約（glob で自動適用、例: `react-components.md`）
- rules/ の内容がルートと一部重複するのは意図的（自己完結性のため）

## エージェント連携パターン

エージェントは単体でも使えるが、以下のパターンで組み合わせると効果的:

### 新機能実装フロー（推奨順序）
1. `tdd` — TDDサイクルで機能実装
2. `a11y-auditor` — アクセシビリティ監査・修正
3. `perf-optimizer` — パフォーマンス最適化（必要な場合のみ）
4. `/review` スキル — 最終レビュー

### リファクタリングフロー
1. `test-writer` — テストカバレッジ追加（不足している場合）
2. `refactor` — テスト保護下でリファクタリング
3. `/check-specs` スキル — 仕様適合チェック

### 学習フロー
1. `explainer` — コードの理解
2. 実装作業（手動またはエージェント経由）
3. `mentor` — 変更差分に対する教育的フィードバック

### 並列実行ルール
以下の読み取り専用エージェントは、他のエージェントと並列実行しても安全:
- `explainer`（読み取り専用）
- `mentor`（読み取り専用）

以下は書き込みを行うため、**同時に1つだけ**実行すること:
- `tdd`, `refactor`, `test-writer`, `a11y-auditor`, `perf-optimizer`
