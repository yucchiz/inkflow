# InkFlow PRD

## 1. 機能要件

### 1.1 ドキュメント管理

#### 作成

| 項目 | 仕様 |
|------|------|
| 作成方法 | 一覧画面の新規作成ボタン(FAB) → エディタ画面へ遷移 |
| 初期状態 | タイトル空、本文空 |
| ID生成 | UUID v4（`crypto.randomUUID()`） |

#### 一覧表示

| 項目 | 仕様 |
|------|------|
| 表示順 | 最終更新日時の降順 |
| 表示項目 | タイトル、本文プレビュー（冒頭2行）、最終更新日時 |
| タイトル未入力時 | 「無題のドキュメント」と表示 |
| クリック/タップ | エディタ画面へ遷移 |

#### 削除

| 項目 | 仕様 |
|------|------|
| 操作方法 | 一覧画面で各ドキュメントの削除ボタン（ゴミ箱アイコン） |
| 確認 | 確認ダイアログを表示（「このドキュメントを削除しますか？」） |
| 確定後 | IndexedDBから完全削除。復元不可 |

#### 空ドキュメント自動削除

| 項目 | 仕様 |
|------|------|
| 条件 | タイトル・本文ともに未入力のままエディタを離脱 |
| 動作 | IndexedDBから自動削除（削除トーストは表示しない） |

### 1.2 エディタ

#### テキスト入力

| 項目 | 仕様 |
|------|------|
| 入力形式 | プレーンテキスト（装飾なし） |
| タイトル入力 | 専用のタイトル入力欄。プレースホルダー：「タイトル」 |
| 本文入力 | `<textarea>`使用。プレースホルダー：「書き始める...」 |
| フォーカス | 画面遷移時に本文入力欄へ自動フォーカス |
| 文字数制限 | なし |

#### 自動保存

| 項目 | 仕様 |
|------|------|
| デバウンス | 500ms（最後の入力から500ms後にIndexedDBへ書き込み） |
| 保存対象 | タイトル、本文、更新日時 |
| インジケーター | ヘッダーに「保存済み」表示 → 2秒後にフェードアウト |
| エラー時 | 「保存に失敗しました」のトースト通知 |

#### 文字数カウント

| 項目 | 仕様 |
|------|------|
| 表示位置 | ステータスバー（画面下部） |
| 対象 | 本文のみ（タイトルは含まない） |
| 表示形式 | 「1,234 文字」（リアルタイム更新） |

#### タイポグラフィ

| 項目 | モバイル | デスクトップ |
|------|---------|-------------|
| 本文フォントサイズ | 16px | 18px |
| タイトルフォントサイズ | 22px | 28px |
| 行間 | 1.8 | 2.0 |
| エディタ最大幅 | 100%（左右padding: 16px） | 680px（中央寄せ） |
| フォント | 日本語に最適化（Noto Serif JP / BIZ UDGothic 等、要検討） |

#### 集中モード

| 項目 | 仕様 |
|------|------|
| 起動 | ヘッダーメニュー内「集中モード」ボタン |
| 動作 | ヘッダー・ステータスバーを非表示にし、エディタのみ表示 |
| UI復帰（デスクトップ） | 画面上部にマウス移動 → ヘッダーがスライドイン |
| UI復帰（モバイル） | 画面上部をタップ → ヘッダーが表示 |
| 解除 | 復帰したヘッダーの解除ボタン / `Escape` キー |

### 1.3 エクスポート

| 機能 | 操作 | 仕様 |
|------|------|------|
| クリップボードコピー | メニュー内「コピー」 | 本文全体をコピー。トースト「コピーしました」 |
| テキストダウンロード | メニュー内「ダウンロード」 | `.txt`（UTF-8）。ファイル名: `{タイトル}.txt`（未入力時は `inkflow-{日付}.txt`）。内容: タイトル + 改行2つ + 本文 |

### 1.4 テーマ切替

| 項目 | 仕様 |
|------|------|
| 切替方法 | 一覧画面：ヘッダーにトグルアイコン / エディタ画面：メニュー内に切替項目 |
| 選択肢 | ライト / ダーク / システム追従 の3択 |
| デフォルト | OSのシステム設定に追従（`prefers-color-scheme`） |
| 永続化 | localStorageに保存 |

### 1.5 PWA

| 項目 | 仕様 |
|------|------|
| ホーム画面追加 | Web App Manifest。`standalone`モード。アイコン 192x192, 512x512 |
| オフライン対応 | Service Worker（Workbox）でアプリシェルをキャッシュ。完全オフライン動作 |
| 更新通知 | 新バージョン検知時に「アプリを更新」通知を表示 |

### 1.6 設定

| 設定名 | デフォルト | 説明 |
|--------|-----------|------|
| theme | `'system'` | `'light'` / `'dark'` / `'system'` |

> MVP方針: 設定画面は設けず、テーマ切替のみUIに露出。フォントサイズ変更等は Phase 2 以降。

```

  ├── カードクリック → [エディタ  /doc/:id]
  └── 削除ボタン → 確認ダイアログ → 削除 → 一覧に留まる
```

## 4. データ仕様

### 4.1 データモデル

#### Document フィールド詳細

| フィールド | 型 | デフォルト値 | バリデーション |
|-----------|------|-------------|---------------|
| id | string | `crypto.randomUUID()` | UUID v4形式 |
| title | string | `""` | 最大200文字。空文字は一覧で「無題のドキュメント」表示 |
| body | string | `""` | 上限なし（IndexedDB容量に依存） |
| createdAt | string | `new Date().toISOString()` | ISO 8601 |
| updatedAt | string | `new Date().toISOString()` | ISO 8601。title/body変更時に更新 |

#### AppSettings

| フィールド | 型 | デフォルト値 | 説明 |
|-----------|------|-------------|------|
| theme | `'light' \| 'dark' \| 'system'` | `'system'` | カラーテーマ |

### 4.2 IndexedDB設計

| 項目 | 値 |
|------|------|
| データベース名 | `inkflow` |
| バージョン | `1` |

| ストア名 | キーパス | インデックス |
|----------|---------|-------------|
| `documents` | `id` | `updatedAt`（降順ソート用） |

設定（AppSettings）は `localStorage` に保存（キー: `inkflow:theme`）。
理由: 少量のキーバリューデータであり、同期的に読めるため初期表示が速い（FOUC防止）。

### 4.3 データアクセス層（DocumentRepository）

```typescript
interface DocumentRepository {
  getAll(): Promise<Document[]>;                       // 全件取得（updatedAt降順）
  getById(id: string): Promise<Document | undefined>;  // 1件取得
  save(doc: Document): Promise<void>;                  // upsert
  remove(id: string): Promise<void>;                   // 削除
}
```

| 方針 | 詳細 |
|------|------|
| ライブラリ | `idb`（IndexedDBのPromiseラッパー） |
| トランザクション | 各操作は単一トランザクション |
| エラーハンドリング | try-catch + トースト通知 |

### 4.4 状態管理

#### DocumentContext

```typescript
interface DocumentState {
  documents: Document[];
  isLoading: boolean;
}
```

| Action | Payload | 説明 |
|--------|---------|------|
| `LOAD_DOCUMENTS` | `Document[]` | IndexedDBから読み込み |
| `ADD_DOCUMENT` | `Document` | 新規追加 |
| `UPDATE_DOCUMENT` | `Document` | 更新 |
| `DELETE_DOCUMENT` | `{ id: string }` | 削除 |

#### ThemeContext

```typescript
interface ThemeState {
  theme: 'light' | 'dark' | 'system';
  resolvedTheme: 'light' | 'dark';  // systemの場合はOS設定を解決した値
}
```

| Action | Payload | 説明 |
|--------|---------|------|
| `SET_THEME` | `'light' \| 'dark' \| 'system'` | テーマ変更 + localStorage保存 |

### 4.5 データフロー

**ドキュメント作成**: FABクリック → Document生成(UUID) → IndexedDB保存 → ADD_DOCUMENT dispatch → エディタへ遷移 → 本文に自動フォーカス

**自動保存**: 編集 → デバウンス(500ms) → updatedAt更新 → UPDATE_DOCUMENT dispatch → IndexedDB保存 → 「保存済み」表示(2秒後フェードアウト)

**削除**: 削除ボタン → 確認ダイアログ → DELETE_DOCUMENT dispatch → IndexedDB削除 → (エディタの場合)一覧へ遷移

**空ドキュメント削除**: エディタ離脱 → title==="" && body==="" 判定 → 空なら DELETE_DOCUMENT dispatch → IndexedDB削除 → 一覧へ遷移（トーストなし）/ 空でなければ通常遷移

**テーマ切替**: トグルクリック → SET_THEME dispatch → localStorage保存 → CSS変数が即座に切替

**アプリ起動**: localStorage→テーマ即時適用(FOUC防止) → IndexedDB→ドキュメント読み込み(非同期) → LOAD_DOCUMENTS dispatch → 描画

**マイグレーション方針**: MVP段階では明示的なマイグレーション機構は設けない。IndexedDBの `version` + `onupgradeneeded` で管理し、新フィールドは読み込み時にデフォルト値で補完。

## 5. アニメーション仕様

### 5.1 基本方針・共通トークン

| 方針 | 詳細 |
|------|------|
| パフォーマンス | `transform` と `opacity` のみアニメーション |
| アクセシビリティ | `prefers-reduced-motion: reduce` でアニメーション無効化 |
| 実装 | CSS Transitions / Animations。MVPでは `framer-motion` 不要 |

```css
:root {
  --duration-fast: 150ms;
  --duration-normal: 200ms;
  --duration-slow: 300ms;
  --ease-out: cubic-bezier(0.0, 0.0, 0.2, 1);
  --ease-in: cubic-bezier(0.4, 0.0, 1, 1);
  --ease-in-out: cubic-bezier(0.4, 0.0, 0.2, 1);
}

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 5.2 アニメーション一覧

| コンポーネント | 状態 | アニメーション | duration | easing |
|--------------|------|--------------|----------|--------|
| 画面遷移 | 一覧→エディタ | フェードイン（opacity 0→1） | 200ms | ease-out |
| 画面遷移 | エディタ→一覧 | フェードアウト（opacity 1→0） | 150ms | ease-in |
| カードリスト | 初回表示 | フェードイン + スライドアップ（translateY 12px→0）。遅延: 50ms*index（最大10件） | 300ms | ease-out |
| カード | ホバー | 背景色変化 | 200ms | ease-in-out |
| カード | ホバー時削除ボタン | opacity 0→1 | 150ms | — |
| FAB | 初回表示 | スケールイン（scale 0→1） | 300ms | ease-out |
| FAB | ホバー | スケールアップ（scale 1→1.05） | 150ms | — |
| FAB | タップ/クリック | スケールダウン→戻り（1→0.92→1） | 150ms | — |
| テーマ切替 | 切替時 | background-color, color, border-color をトランジション | 300ms | ease-in-out |
| 保存ステータス | 表示 | フェードイン | 150ms | — |
| 保存ステータス | 消滅 | 2秒後にフェードアウト | 300ms | — |
| 集中モード | ヘッダー非表示 | スライドアップ（translateY 0→-100%） | 200ms | ease-in |
| 集中モード | ヘッダー復帰 | スライドダウン（translateY -100%→0） | 200ms | ease-out |
| 集中モード | ステータスバー | フェードアウト/イン | 200ms | — |
| トースト | 表示 | スライドアップ + フェードイン（translateY 16px→0） | 200ms | ease-out |
| トースト | 消滅 | フェードアウト（3秒後に自動開始） | 200ms | ease-in |
| 確認ダイアログ | オーバーレイ | フェードイン | 200ms | — |
| 確認ダイアログ | 本体表示 | スケールイン（scale 0.95→1）+ フェードイン | 200ms | ease-out |
| 確認ダイアログ | 閉じる | フェードアウト | 150ms | ease-in |
| ドロップダウン | 表示 | スケールイン（scale 0.95→1）+ フェードイン。右上起点 | 150ms | ease-out |
| ドロップダウン | 閉じる | フェードアウト | 100ms | ease-in |

テーマ切替の実装: `<body>` に `theme-transitioning` クラスを一時付与し、全要素に transition を適用。
