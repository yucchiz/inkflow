# 03 - 画面仕様

## 画面一覧

| 画面名 | ルート | 説明 |
|--------|--------|------|
| ドキュメント一覧 | `/` | メイン画面。ドキュメントのリスト表示 |
| エディタ | `/doc/:id` | ドキュメントの執筆・編集 |

### ヘッダー

| 要素 | 仕様 |
|------|------|
| アプリ名 | 「InkFlow」をブランドフォントで表示 |
| テーマ切替 | ☀（ライト）/ 🌙（ダーク）のトグルアイコン |
| 位置 | 画面上部に固定（sticky） |

### ドキュメントリスト

| 要素 | 仕様 |
|------|------|
| リスト実装 | 仮想スクロールは不要（MVP。100件程度で十分な性能） |
| 表示順 | 最終更新日時の降順 |
| 最大幅 | 640px（中央寄せ、左右に余白） |

### ドキュメントカード

| 要素 | 仕様 |
|------|------|
| タイトル | 1行。はみ出し時は省略（...）。未入力時は「無題のドキュメント」 |
| プレビュー | 本文の冒頭2行分。はみ出し時は省略（...） |
| 更新日時 | 右下に小さく表示（日時フォーマットは後述） |
| クリック/タップ | エディタ画面へ遷移 |
| ホバー（デスクトップ） | 背景色が微かに変化。削除ボタンが表示される |
| 削除ボタン | ホバー時（デスクトップ）またはカード右端に常時表示（モバイル）。ゴミ箱アイコン |
| 削除操作 | 削除ボタンクリック → 確認ダイアログ → 削除確定 |

### 日時表示フォーマット

| 条件 | 表示例 |
|------|--------|
| 今日 | `12:30` （時刻のみ） |
| 昨日 | `昨日` |
| 今年内 | `3月1日` |
| 去年以前 | `2025年3月1日` |

### FAB（新規作成ボタン）

| 要素 | 仕様 |
|------|------|
| 位置 | 画面右下（固定配置） |
| デザイン | 円形、アクセントカラー、＋アイコン |
| クリック/タップ | 新規ドキュメントを作成し、エディタ画面へ遷移 |
| デスクトップ代替 | FABに加え、ヘッダー付近にもテキストリンク「新規作成」を表示（検討） |

### 空状態（EmptyState）

```
┌──────────────────────────────┐
│  InkFlow              [☀/🌙] │
├──────────────────────────────┤
│                              │
│                              │
│                              │
│         ✏️                    │
│    まだドキュメントがありません │
│    ＋ボタンで書き始めましょう   │
│                              │
│                              │
│                              │
│                         [＋] │
└──────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| アイコン | ペンまたはドキュメントのアイコン |
| テキスト | 「まだドキュメントがありません」+「＋ボタンで書き始めましょう」 |
| 表示条件 | ドキュメントが0件のとき |

## 2. エディタ画面（`/doc/:id`）

### レイアウト — モバイル

```
┌──────────────────────────────┐
│  [←]          保存済み [···] │  ← ヘッダー（戻る + ステータス + メニュー）
├──────────────────────────────┤
│                              │
│  ブログ記事の下書き           │  ← タイトル入力欄
│                              │
│  ──────────────────────────  │  ← 区切り線（視覚的な分離）
│                              │
│  今日はReactのhooksについて   │  ← 本文入力欄
│  書いてみようと思います。     │
│                              │
│  useStateは最も基本的な       │
│  フックで、コンポーネントに   │
│  ローカルな状態を持たせる     │

└──────────────────────────────┘
```

### レイアウト — デスクトップ

### ヘッダー

| 要素 | 仕様 |
|------|------|
| 戻るボタン | 左端。← アイコン。クリックで一覧画面へ遷移 |
| 保存ステータス | 中央付近。保存完了時に「保存済み」を表示 → 2秒後にフェードアウト |
| メニューボタン | 右端。[···] アイコン。クリックでドロップダウンメニュー表示 |

### メニュー（ドロップダウン）

| メニュー項目 | 動作 |
|-------------|------|
| コピー | 本文全体をクリップボードにコピー。トースト「コピーしました」表示 |
| ダウンロード | .txt ファイルとしてダウンロード |
| 削除 | 確認ダイアログ → 削除 → 一覧画面へ遷移 |

### タイトル入力欄

| 要素 | 仕様 |
|------|------|
| フォント | 本文より大きいサイズ（モバイル22px / デスクトップ28px）、太字ウェイト |
| プレースホルダー | 「タイトル」（薄いグレーで表示） |
| 改行 | 不可（Enterキーで本文入力欄へフォーカス移動） |
| ボーダー | なし（アウトラインなしのクリーンな入力欄） |

### 本文入力欄

| 要素 | 仕様 |
|------|------|
| 実装 | `<textarea>` を使用（contentEditableは使わない） |
| フォント | 日本語に最適化されたフォント（モバイル16px / デスクトップ18px） |
| 行間 | 1.8〜2.0 |
| 最大幅 | 680px（中央寄せ） |
| 高さ | コンテンツに合わせて自動拡張（auto-resize） |
| プレースホルダー | 「書き始める...」 |
| ボーダー | なし |
| フォーカス | 新規作成時は自動フォーカス |
| スクロール | ページ全体がスクロール（textarea内スクロールではない） |

### ステータスバー

| 要素 | 仕様 |
|------|------|
| 位置 | 画面下部に固定（sticky bottom） |
| 内容 | 文字数カウント（「1,234 文字」の形式） |
| デザイン | 控えめな背景色、小さなフォントサイズ |
| 集中モード時 | 非表示 |

### 集中モード

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│                                                      │
│              ブログ記事の下書き                        │
│                                                      │
│              ────────────────────                    │
│                                                      │
│              今日はReactのhooksについて                │
│              書いてみようと思います。                   │
│                                                      │
│              useStateは最も基本的な                    │
│              フックで、コンポーネントに                 │
│              ローカルな状態を持たせる                   │
│              ことができます。|                         │
│                                                      │
│                                                      │
│                                                      │
│                                                      │
└──────────────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| 起動方法 | ヘッダーメニュー内「集中モード」ボタン |
| 動作 | ヘッダーとステータスバーを非表示。本文のみの没入体験 |
| UI復帰（デスクトップ） | 画面上部にマウスカーソルを移動 → ヘッダーがスライドイン |
| UI復帰（モバイル） | 画面上部をタップ → ヘッダーが表示 |
| 解除 | 復帰したヘッダーの集中モード解除ボタン、または `Escape` キー |

## 3. 共通UI要素

### トースト通知

| 要素 | 仕様 |
|------|------|
| 位置 | 画面下部中央 |
| 表示時間 | 3秒（自動消滅） |
| アニメーション | 下からフェードイン → フェードアウト |
| 用途 | 「コピーしました」「保存に失敗しました」等 |

### 確認ダイアログ

| 要素 | 仕様 |
|------|------|
| 表示方法 | モーダルオーバーレイ |
| 用途 | ドキュメント削除の確認 |
| ボタン | 「キャンセル」（左）+「削除」（右、赤系アクセント） |
| 閉じる方法 | キャンセルボタン / オーバーレイクリック / Escapeキー |

## 4. 画面遷移図

```
[アプリ起動（PWA / ブラウザ）]
    │
    └──→ [ドキュメント一覧  /]
              │
              ├── FABクリック
              │   → 新規ドキュメント作成（IndexedDB）
              │   → [エディタ  /doc/:id] へ遷移
              │
              ├── ドキュメントカードクリック
              │   → [エディタ  /doc/:id] へ遷移
              │
              └── 削除ボタンクリック
                  → 確認ダイアログ
                  → 削除確定 → ドキュメント削除
                  → 一覧画面に留まる

[エディタ  /doc/:id]
    │
    ├── 戻るボタン / ブラウザバック
    │   → [ドキュメント一覧  /] へ遷移
    │
    ├── メニュー → コピー
    │   → クリップボードにコピー → トースト表示
    │
    ├── メニュー → ダウンロード
    │   → .txt ファイルダウンロード
    │
    └── メニュー → 削除
        → 確認ダイアログ
        → 削除確定 → [ドキュメント一覧  /] へ遷移
```

### 遷移アニメーション

| 遷移 | アニメーション |
|------|--------------|
| 一覧 → エディタ | フェードイン（200ms）。スライドではなくシンプルに |
| エディタ → 一覧 | フェードアウト（150ms） |
| ダイアログ表示 | オーバーレイのフェードイン + ダイアログのスケールイン |
| ダイアログ閉じ | フェードアウト |
