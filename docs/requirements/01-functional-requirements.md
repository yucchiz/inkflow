# 01 - 機能要件

## 1. ドキュメント管理

### 1.1 ドキュメントの作成

| 項目 | 仕様 |
|------|------|
| 作成方法 | 一覧画面の新規作成ボタンをクリック → エディタ画面へ遷移 |
| 初期状態 | タイトル空、本文空 |
| ID生成 | UUID v4（`crypto.randomUUID()`） |
| 保存方式 | 自動保存（後述） |
| 空ドキュメントの扱い | タイトル・本文ともに未入力のままエディタを離れた場合、自動削除 |

### 1.2 ドキュメントの一覧表示

| 項目 | 仕様 |
|------|------|
| 表示順 | 最終更新日時の降順（新しいものが上） |
| 表示項目 | タイトル、本文プレビュー（冒頭2行程度）、最終更新日時 |
| タイトル未入力時 | 「無題のドキュメント」と表示 |
| クリック/タップ | エディタ画面へ遷移 |

### 1.3 ドキュメントの削除

| 項目 | 仕様 |
|------|------|
| 操作方法 | 一覧画面で各ドキュメントの削除ボタン（ゴミ箱アイコン） |
| 確認 | 確認ダイアログを表示（「このドキュメントを削除しますか？」） |
| 確定後 | IndexedDBから完全削除。復元不可 |

### 1.4 ドキュメントの上限

| 項目 | 仕様 |
|------|------|
| 最大保存件数 | MVP段階では制限なし（IndexedDBの容量に依存） |
| 注意 | IndexedDBのブラウザ別容量制限について非機能要件で定義 |

## 2. エディタ

### 2.1 テキスト入力

| 項目 | 仕様 |
|------|------|
| 入力形式 | プレーンテキスト（装飾・マークアップなし） |
| タイトル入力 | エディタ上部に専用のタイトル入力欄 |
| 本文入力 | タイトル欄の下に本文入力エリア |
| 文字数制限 | なし |
| 改行 | 自由に改行可能 |
| プレースホルダー | タイトル欄：「タイトル」 / 本文欄：「書き始める...」 |
| フォーカス | エディタ画面遷移時に本文入力欄へ自動フォーカス |

### 2.2 自動保存

| 項目 | 仕様 |
|------|------|
| トリガー | テキスト変更のたびに自動保存 |
| デバウンス | 500ms（最後の入力から500ms後にIndexedDBへ書き込み） |
| 保存対象 | タイトル、本文、更新日時 |
| 保存インジケーター | ヘッダーに「保存済み」の小さな表示（保存完了時に一瞬表示） |
| エラー時 | 「保存に失敗しました」のトースト通知 |

### 2.3 文字数カウント

| 項目 | 仕様 |
|------|------|
| 表示位置 | エディタ画面下部のステータスバー |
| 表示内容 | 本文の文字数（タイトルは含まない） |
| 更新タイミング | リアルタイム（入力のたびに更新） |
| 表示形式 | 「1,234 文字」 |

### 2.4 タイポグラフィ

| 項目 | 仕様 |
|------|------|
| 本文フォント | 日本語に美しいフォント（Noto Serif JP or BIZ UDGothic 等、要検討） |
| タイトルフォント | 本文と同一ファミリー、ウェイト違い |
| フォントサイズ（本文） | 16px〜18px（デバイスによる調整） |
| フォントサイズ（タイトル） | 24px〜28px |
| 行間 | 1.8〜2.0（読みやすさ重視） |
| 最大幅 | 680px程度（長い行を防ぎ、読みやすさを確保） |
| 水平配置 | 中央寄せ（左右に十分な余白） |

### 2.5 集中モード

| 項目 | 仕様 |
|------|------|
| 起動方法 | エディタ画面のヘッダーにある集中モードボタン |
| 動作 | ヘッダー・ステータスバーを非表示にし、エディタのみ表示 |
| 解除方法 | 画面上部にマウスを移動（デスクトップ）/ 画面タップ（モバイル）でUIが再表示 |
| キーボードショートカット | `Escape` で集中モード解除 |

## 3. エクスポート

### 3.1 クリップボードコピー

| 項目 | 仕様 |
|------|------|
| 操作方法 | エディタ画面のヘッダーメニュー内「コピー」ボタン |
| コピー内容 | 本文のプレーンテキスト全体 |
| フィードバック | 「コピーしました」のトースト通知 |

### 3.2 テキストファイルダウンロード

| 項目 | 仕様 |
|------|------|
| 操作方法 | エディタ画面のヘッダーメニュー内「ダウンロード」ボタン |
| ファイル形式 | `.txt`（UTF-8） |
| ファイル名 | `{タイトル}.txt`（タイトル未入力時は `inkflow-{日付}.txt`） |
| 内容 | タイトル + 改行2つ + 本文 |

---

## 4. テーマ（ダークモード / ライトモード）

### 4.1 テーマ切替

| 項目 | 仕様 |
|------|------|
| 切替方法 | ヘッダーまたは一覧画面にテーマ切替トグル |
| デフォルト | OSのシステム設定に追従（`prefers-color-scheme`） |
| 選択肢 | ライト / ダーク / システム追従 の3択 |
| 永続化 | ユーザーの選択をlocalStorageに保存 |
| 切替アニメーション | スムーズなカラートランジション |

---

## 5. PWA

### 5.1 ホーム画面追加

| 項目 | 仕様 |
|------|------|
| 対応 | Web App Manifest による「ホーム画面に追加」 |
| アプリ名 | InkFlow |
| 表示モード | `standalone`（ブラウザUIなしで起動） |
| アイコン | 192x192, 512x512 の PWA アイコン |
| テーマカラー | ブランドカラーに合わせて設定 |
| 起動画面 | スプラッシュスクリーン表示 |

### 5.2 オフライン対応

| 項目 | 仕様 |
|------|------|
| 動作 | 完全オフラインで全機能が動作 |
| キャッシュ戦略 | アプリシェル（HTML/CSS/JS）をService Workerでキャッシュ |
| データ | IndexedDBに保存されているため、オフラインでも読み書き可能 |
| 更新通知 | 新バージョンがある場合「アプリを更新」の通知を表示 |

---

## 6. 設定

### 6.1 設定項目

| 設定名 | デフォルト値 | 説明 |
|--------|------------|------|
| theme | `'system'` | テーマ（`'light'` / `'dark'` / `'system'`） |

> **MVP方針**: 設定項目は最小限にする。設定画面は設けず、テーマ切替のみUIに露出する。
> フォントサイズ変更、行間変更等は Phase 2 以降で検討する。

## 7. 将来機能（MVP対象外）

以下はMVPには含めないが、将来的に追加を検討する機能。

| 機能 | 概要 | Phase |
|------|------|-------|

| クラウド同期 | Supabase等を利用したデバイス間同期 | Phase 3 |
| Markdownプレビュー | オプションでMarkdownをプレビュー表示 | Phase 3 |
| AI執筆アシスタント | AI連携による文章校正・提案 | Phase 3 |
