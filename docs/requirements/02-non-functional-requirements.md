## 3. PWA要件

### Web App Manifest

| 項目 | 値 |
|------|------|
| name | InkFlow |
| short_name | InkFlow |
| display | standalone |
| orientation | any |
| start_url | / |
| background_color | ライトテーマの背景色 |
| theme_color | ブランドカラー |
| icons | 192x192 (maskable), 512x512 (maskable) |

### Service Worker

| 項目 | 仕様 |
|------|------|
| 生成方法 | Vite PWA Plugin (Workbox) による自動生成 |
| キャッシュ戦略 | アプリシェル: CacheFirst / フォント: CacheFirst |
| プリキャッシュ | HTML, CSS, JS, フォントファイル |
| 更新検知 | 新バージョンデプロイ時に更新通知を表示 |
| 更新操作 | ユーザーが「更新する」をクリック → ページリロード |

### オフライン動作

| 項目 | 仕様 |
|------|------|
| 初回アクセス | オンライン必須（アセットのダウンロード） |
| 2回目以降 | 完全オフラインで動作 |
| データ | IndexedDBに保存。オフラインでも読み書き可能 |
| オフライン表示 | 特別なオフライン画面は不要（通常通り動作するため） |

## 4. パフォーマンス

### Core Web Vitals 目標

| 指標 | 目標値 | 説明 |
|------|--------|------|
| LCP (Largest Contentful Paint) | < 2.5s | 最大コンテンツの描画完了 |
| FID (First Input Delay) | < 100ms | 初回入力の応答速度 |
| CLS (Cumulative Layout Shift) | < 0.1 | レイアウトのズレ |

### アプリケーション性能

| 項目 | 基準値 |
|------|--------|
| 初回ロード | 3秒以内（フォントロード含む、3G回線想定） |
| PWA起動 | 1秒以内（キャッシュ済み状態） |
| エディタ入力遅延 | 体感0ms（入力とレンダリングの遅延なし） |
| 自動保存 | デバウンス500ms → IndexedDB書き込み完了 100ms以内 |
| 一覧画面表示 | 100件のドキュメントを60fpsでスクロール |
| バンドルサイズ | gzip後 200KB以下（初期ロード分） |

### 最適化方針

| 手法 | 詳細 |
|------|------|
| コード分割 | ルートベースの遅延ロード（一覧 / エディタ） |
| フォント最適化 | `font-display: swap`、サブセット化（日本語フォントの軽量化） |
| 画像最適化 | アイコンはSVGを使用。ラスタ画像は最小限 |
| React最適化 | `React.memo`, `useMemo`, `useCallback` を適切に使用 |

## 5. データ永続化

| 項目 | 仕様 |
|------|------|
| メインストレージ | IndexedDB（`idb`経由） |
| 設定ストレージ | localStorage（テーマ設定等の軽量データ） |
| オフライン対応 | 完全オフラインで動作（ネットワーク依存なし） |
| クラウド同期 | なし（MVP） |
| データバックアップ | ユーザーによる手動エクスポート（.txtダウンロード） |

### IndexedDB 容量

| ブラウザ | 上限目安 |
|---------|---------|
| Chrome | ディスク空き容量の最大80% |
| Safari | 1GB（ユーザー許可で拡張可能） |
| Firefox | ディスク空き容量の最大50% |

> プレーンテキストのみの保存のため、通常使用で容量問題が発生する可能性は極めて低い。

## 6. セキュリティ

| 項目 | 仕様 |
|------|------|
| HTTPS | 必須（PWA要件。ホスティングサービスが自動提供） |
| 認証 | なし（ローカルデータのみ） |
| 個人情報収集 | なし |
| 外部通信 | なし（フォントCDNへの初回アクセスのみ） |
| XSS対策 | Reactのデフォルトエスケープ。`dangerouslySetInnerHTML` 不使用 |
| CSP | Content Security Policy ヘッダーを設定 |

---

## 7. アクセシビリティ

| 項目 | 仕様 |
|------|------|
| セマンティックHTML | 適切なHTML要素の使用（`<main>`, `<nav>`, `<article>` 等） |
| キーボード操作 | 全機能がキーボードのみで操作可能 |
| フォーカス管理 | フォーカスリングの明示、画面遷移時の適切なフォーカス移動 |
| カラーコントラスト | WCAG AA 準拠（テキスト 4.5:1 以上、大文字 3:1 以上） |
| スクリーンリーダー | 適切な `aria-label`, `aria-live` の使用 |
| Reduce Motion | `prefers-reduced-motion` メディアクエリでアニメーション無効化に対応 |
