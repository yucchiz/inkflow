# InkFlow — 書くことに集中する、執筆アプリ

## App Concept

**コンセプト**: 「書く。ただ、それだけを美しく。」

ブログ記事やエッセイを書く人のための、プレーンテキスト執筆アプリ。
装飾も設定もない。美しいUIと心地よい書き味だけがある。
ブラウザで開いて、すぐ書き始める。PWA対応で、ホーム画面からいつでも。

---

## 1. 解決する課題

### 書く人が感じている問題

| 課題 | 詳細 |
|------|------|
| 機能過多 | Notion, Google Docsは高機能すぎて「書く」に集中できない |
| 見た目の退屈さ | メモ帳やテキストエディタは機能的だが、使いたいと思えるUIではない |
| 環境依存 | ネイティブアプリはデバイスに縛られる。同期設定も面倒 |
| 起動の重さ | 書きたいと思った瞬間に、すぐ書き始められるツールがない |

### InkFlowの回答

> **「余計なものを削ぎ落とした先にある、書く行為そのものの心地よさ」**

- 機能は最小限。プレーンテキストだけ
- UIはおしゃれで洗練されている
- ブラウザで即起動。PWAでネイティブアプリのように使える
- オフライン完結。ネットワーク不要

---

## 2. ターゲットユーザー

### プライマリ：ブログ・記事ライター

- ブログ記事の下書きを気軽に書きたい人
- アイデアを長文で書き出す場所がほしい人
- シンプルなツールで書くことに集中したい人
- おしゃれな執筆環境にモチベーションを感じる人

### ユーザーの行動パターン

```
思いつく → InkFlowを開く → 書く → 保存（自動）→ 仕上がったらコピーしてブログに貼る
```

## 3. 競合分析 & ポジショニング

### 既存の競合

| アプリ名 | 特徴 | InkFlowとの違い |
|----------|------|-----------------|
| iA Writer | ミニマルなMarkdownエディタ | 有料ネイティブアプリ。Markdown前提 |
| Hemingway Editor | 文章の改善提案 | 英語圏向け。編集支援が主目的 |
| Draft | ブラウザベースの執筆ツール | 機能が多い。UIが古い |
| Notion | 万能ワークスペース | 高機能すぎる。書くことへの集中を阻害 |
| Google Docs | ドキュメント作成 | 共同編集前提。書く体験としては凡庸 |
| メモ帳 / Notes | OS標準メモ | 機能的だがUIに魅力がない |

### InkFlowのポジション

```
高機能 ← ────────────────────── → シンプル
                                       ★ InkFlow
Notion ── Google Docs ── iA Writer ──

UIの魅力なし ← ──────────────── → おしゃれ
                                       ★ InkFlow
メモ帳 ── Draft ── Notion ─────────

ネイティブ専用 ← ────────────── → どこでも使える
                                       ★ InkFlow
iA Writer ── Apple Notes ──────────
```

**差別化ポイント**:
- **PWA × プレーンテキスト × 美しいUI** の組み合わせが競合不在
- 無料でブラウザから使える手軽さ
- 「書く道具」としてのシンプルさと、「使いたくなる」デザインの両立

## 4. 機能サマリー（MVP）

### 4.1 ドキュメント管理

- ドキュメントの新規作成・一覧表示・削除
- タイトルと本文の自動分離（1行目 = タイトル）
- 最終更新日時でのソート
- 自動保存（入力のたびに即座に保存）

### 4.2 エディタ

- プレーンテキスト入力（装飾なし）
- フルスクリーン集中モード
- 文字数カウント（リアルタイム表示）
- 美しいタイポグラフィ（フォント・行間・余白の最適化）
- レスポンシブ対応（モバイル / デスクトップ）

### 4.3 エクスポート

- クリップボードにコピー
- .txt ファイルとしてダウンロード

### 4.4 PWA

- ホーム画面に追加（Add to Home Screen）
- オフライン完全動作
- ネイティブアプリライクな起動体験

### 4.5 デザイン

- シンプルでおしゃれなUI
- ダークモード / ライトモード対応
- 心地よいアニメーションとトランジション
- 日本語に最適化されたタイポグラフィ

## 5. 技術スタック

```
Framework:      React + Vite（SPA）
Styling:        CSS Modules or Tailwind CSS
State:          Zustand（軽量状態管理）
Storage:        IndexedDB（Dexie.js経由）
PWA:            Vite PWA Plugin（Workbox）
Font:           Google Fonts（Noto Sans JP + エディタ用フォント）
Deploy:         Vercel or Cloudflare Pages（静的ホスティング）
Language:       TypeScript
```

### 技術選定の理由

| 選択 | 理由 |
|------|------|
| Vite (SPA) | SSR不要。高速ビルド。PWA対応が容易 |
| Zustand | Context + useReducerより簡潔。ボイラープレート最小 |
| IndexedDB (Dexie.js) | localStorageより大容量。長文の保存に適する |
| Tailwind CSS | 迅速なUI構築。デザインの一貫性を保ちやすい |
| Vite PWA Plugin | Service Worker生成を自動化 |

## 6. 画面構成

```
[アプリ起動]
    │
    └──→ [ドキュメント一覧]
              │
              ├── 新規作成 ──→ [エディタ（新規）]
              │                      │
              │                      └── 戻る ──→ [ドキュメント一覧]
              │
              └── ドキュメント選択 ──→ [エディタ（既存）]
                                       │
                                       └── 戻る ──→ [ドキュメント一覧]
```

- **たった2画面**: ドキュメント一覧 + エディタ
- シンプルな画面遷移、迷わないUI

## 8. デザイン方針

### 原則

1. **余白を活かす** — 情報密度を下げ、書くことに集中できる空間を作る
2. **タイポグラフィファースト** — フォント・行間・文字サイズの最適化が最優先
3. **控えめなインタラクション** — アニメーションはさりげなく、邪魔にならない程度に
4. **一貫性** — 色、余白、角丸、影のルールを統一する

### カラーパレット（方針）

| モード | 背景 | テキスト | アクセント |
|--------|------|---------|-----------|
| ライト | 温かみのある白 | 落ち着いたダークグレー | 控えめなブランドカラー |
| ダーク | 深いダークグレー | 柔らかなオフホワイト | 同上 |

### タイポグラフィ（方針）

| 用途 | 方針 |
|------|------|
| エディタ本文 | 日本語に美しいフォント。十分な行間（1.8〜2.0）。最適な文字サイズ |
| UI要素 | クリーンなサンセリフ体 |
| 文字幅 | エディタは最大幅を制限し、読みやすい行長を維持（60〜80文字程度） |

## まとめ

InkFlowは「書くことだけ」に特化したアプリ。

機能の少なさは弱点ではなく、**設計思想**。
おしゃれなUIと心地よい書き味で、
「また書きたい」と思わせる執筆体験を提供する。

**「何ができるか」ではなく「どう書けるか」で選ばれるアプリ。**
